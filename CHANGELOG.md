## Version Format

Documentation version: `[specification-version].[documentation-patch].[hotfix]`

Example: `2.6.0` = Specification v2.6, documentation generation 0, no hotfixes

### Version Number Components

- **Major** (specification-version): Matches the official specification version
- **Minor** (documentation-patch): Incremented when documentation is regenerated
- **Patch** (hotfix): Incremented for documentation bug fixes only
